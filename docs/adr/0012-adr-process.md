# ADR-0012: ADR process
Status: Accepted

## Context
The project will grow; decisions must remain discoverable and revisable.

## Decision
Record every significant architectural decision as a numbered ADR in
`docs/adr/` (Context → Decision → Consequences, with a Status). Superseding a
decision adds a new ADR and marks the old one Superseded.

## Consequences
- Shared, durable rationale; easier onboarding and safer change.
